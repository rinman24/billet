01 The idea

Everything runs on one metaphor: a posting.

In its truest sense a billet is an assignment of quarters. A Host has berths; billet posts each repo into one. The identity is that rack — three berths posted, three left open.

Host
The shared cloud machine. It has a fixed set of berths.
Berth
One slot on the Host where a Workspace can be posted.
Workspace
Your repo's devcontainer, running inside a berth.
Post
To billet a repo — claim a berth and bring it up.
02 The mark

The berth rack.

Two columns, three rows. Occupied berths carry the accents; open berths are outlined. It is square-first, so it drops straight into a favicon or an app icon.

07 The command line

The terminal is where the brand lives.

A magenta prompt. Status carried by color, not adjectives. Commands that read like a sentence: billet post api.

host — billet
❯ billet post api web
  api   posting… → berth 3
  web   posting… → berth 5
❯ billet ls
apirunningberth 3
webbuildingberth 5
docsopenberth 1
❯
Command surface
billet post <repo>
Claim a berth on the Host and bring the Workspace up.
billet ls
List berths — running, building, or open.
billet shell <repo>
Drop into the Workspace posted at a berth.
billet logs <repo>
Stream a Workspace's build and run output.
billet rm <repo>
Vacate the berth and free it on the Host.
Voice
Terse and lowercase. No exclamation.
Present tense — posting, not will post.
Status is a color, then a word.
